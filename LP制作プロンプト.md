## 目的・世界観

「ただのプロフィールLP」ではなく、**"人生の世界観を体験するドキュメンタリー型LP"** を制作してください。

**Netflixドキュメンタリー × SNS没入体験 × エッセイ × モバイル縦読みLP** の世界観。

目的は「申し込み」よりも先に、**この人の人生・価値観・思想・感情に没入し、共感と信頼を作ること**。

読了後に「この人に会いたい」「話してみたい」「この人の価値観が好き」と思ってもらう構成。

最重要テーマ：**"生まれてきてよかった"を思い出す旅**

## レイアウト設計

- **スマホ最優先**（max-width: 520px 中央寄せ）
- 縦スクロール没入型
- 文字量は多くてよい。ただし**"長いのに読める"構造**にすること

### レイアウトルール
1. 1セクション1メッセージ
2. 余白を多く取る（padding: 80px 28px 以上）
3. 感情単位でセクションを切る
4. 長文は分解して短い行に
5. 大きな見出し → 短文 → 余白 の繰り返し
6. 「続きを読みたくなる設計」を最重視

### SECTION 1｜ファーストビュー（スクロール連動アニメーション）

**実装方法：GSAP ScrollTrigger 推奨。使わない場合は IntersectionObserver + CSS transition。**

セクション全体を `position: sticky` または ScrollTrigger の `pin` で固定し、
スクロール量に応じてテキスト・写真・背景が変化する。

#### 表示ステップ（スクロールに連動）

**STEP 1（最初の画面）**
- 背景：暗め（#09040c 系）
- 中央にタイポグラフィのみ
- 「呼吸」アニメーション：ごくわずかに明度が揺れる（opacityが0.85〜1.0を9秒周期で往復）
- 表示テキスト：
  ```
  わたしは、
  生まれてきて
  よかった。
  ```

**STEP 2（スクロール開始）**
- 上のテキストがフェードアウト
- 続きが浮かび上がる：
  ```
  そう思えなかった私が、
  人生をかけて、
  探し続けたもの。
  ```

**STEP 3（写真浮かび上がり）**
- 本人写真（`assets/hero.png`）がゆっくり現れる
- 最初は `blur(20px) opacity(0)` → スクロールで `blur(0) opacity(0.75)` に変化
- PC：右寄せ。スマホ：背景気味（左右にはみ出ても可）
- テキスト：
  ```
  愛されること。
  認められること。
  ```

**STEP 4**
```
何度も傷ついて、
何度も迷って、
そして辿り着いた。
```

**STEP 5（感情の着地）**
- フォントサイズを少し大きく
```
"本当の自分"に
還ること。
```

**STEP 6（締め）**
- 小さめのフォント
```
これは、
私の人生の記録。
```
- その下に静かなスクロール誘導：`scroll to read` または `このまま読む`（ボタンではなく小さいテキスト）

#### 背景に浮かぶキーワード（薄く・一瞬）
以下のワードをランダムに `opacity: 0.04〜0.08` 程度で背景に表示。メインテキストの邪魔をしない。
`親の離婚` `孤独` `承認欲求` `恋愛` `結婚` `離婚` `子育て` `旅` `ヒーリング` `自分に還る`

#### チャプター表示（画面右端に固定）
スクロールに連動して章番号と章名を小さく表示：
```
01 PROLOGUE → 02 MEMORY → 03 WOUND → 04 JOURNEY → 05 RETURN
```

## アニメーション実装仕様

### ファーストビュー（GSAP ScrollTrigger）
```javascript
// ファーストビュー全体をpin
ScrollTrigger.create({
  trigger: ".hero",
  start: "top top",
  end: "+=300%",
  pin: true,
  scrub: 1
})

// 各テキストブロックをスクロール量で出現
// 写真は blur(20px)→blur(0) + opacity(0)→0.75 で "思い出される" 演出
// 背景キーワードはタイマーでランダム表示（opacity 0.05程度）
```

### 通常セクション（IntersectionObserver）
- `.reveal` クラス：`opacity: 0; transform: translateY(28px)` → `is-visible` で解除
- threshold: 0.18
- 遅延付きで子要素を順番に出現

### チャプターインジケーター
- 画面右端に固定（`position: fixed; right: 16px`）
- 現在のセクションに応じてチャプター番号・名前を更新
- 小さく・薄く。装飾ではなく「気配」として

### 禁止事項
- 動きを速くしない
- 派手なトランジションなし
- パーティクルや過度なグロー演出なし

## 注意事項・優先順位

1. **読みやすさ最優先**。アニメーションは「読ませるための間」であり装飾ではない
2. スマホで実際にスクロールして違和感がないこと
3. テキストは提供されたコピーを忠実に使う（勝手に書き換えない）
4. CTAは最後に1箇所のみ。売り込み感NG
5. 1ファイルで完結させること（CDN読み込みは可）
6. 後から各セクションのテキストを編集しやすい構造にすること
